## 1. 文档目标与背景

### 1.1 目标

基于业务包开发方案文档和 PRD 定义的产品愿景，结合当前系统界面现状，输出一个系统性的界面修改开发方案，明确：
- 哪些页面需要新增
- 哪些页面需要增强
- 增强的具体功能点
- 与业务包方案的对应关系

### 1.2 背景

根据 [可扩展Agent平台-通用PRD-v1.0.md](../可扩展Agent平台-通用PRD-v1.0.md) §11 信息架构与页面建议，平台需要以下页面：

| 页面 | PRD定义目标 | 当前状态 |
|------|------------|---------|
| 工作台首页 | 常用入口、最近会话、已启用业务包、待审批事项 | ✅ 已有基础，需增强 |
| 统一对话页 | 三区布局（会话列表、对话区、上下文面板） | ⚠️ 仅有对话演练场，缺会话列表和上下文面板 |
| 结果详情页 | 完整结果与依据展示 | ❌ 缺失 |
| 审批确认页 | 动作摘要、风险说明、审批操作 | ✅ 已有基础 |
| 业务包与插件管理页 | 业务包列表、插件健康、版本管理、发布/回滚 | ⚠️ 已有列表，缺能力契约和发布编排 |
| 评估与运营看板页 | 总览指标、业务包表现、插件排行、成本趋势 | ❌ 缺失 |
| 开发者体验页 | 能力目录、插件调试 Playground | ❌ 缺失 |
| 个人记忆管理页 | 长期记忆查看、编辑、导出、删除 | ❌ 缺失 |

### 1.3 与业务包方案的关系

根据 [行业业务包-通用开发方案.md](业务包相关开发方案/行业业务包-通用开发方案.md) §15.2 前端任务拆解，业务包对前端的具体要求：

- 工作台增加「业务包切换」入口
- 草稿包展示（一次提交多张草稿，支持一键确认）
- 审批抽屉（金额/工时阈值可视化）
- 引用卡片支持 SOP 步骤号、安全标识、历史工单链接
- 安全提示组件（强制置顶展示）

根据 [制造业设备运维助手-业务包开发文档.md](业务包相关开发方案/制造业设备运维助手-业务包开发文档.md) §16.2 前端任务拆解：

- 草稿动作卡片 UI（含风险等级标签）
- 审批确认页（含动作摘要、参数明细、影响范围）
- 引用卡片支持 SOP 步骤号、安全标识、历史工单链接

#### 4.1.5 与业务包的对应

根据 [制造业设备运维助手-业务包开发文档.md](业务包相关开发方案/制造业设备运维助手-业务包开发文档.md) §16.1：

**制造业故障处置剧本响应结构**：
```json
{
  "answer": "...安全提示：先泄压再开盖...故障码 E-204 = 液压压力低...",
  "sources": [
    {"source": "equipment_manual", "title": "冲压机液压系统-故障码表", "version": "v3.2", "locator": "ch5.4"},
    {"source": "safety_regulation", "title": "液压设备维修安全规程", "version": "2025", "locator": "§3.1"},
    {"source": "workorder_kb", "title": "WO-2026-00821 同类故障处置", "locator": "wo:00801"}
  ],
  "draft_actions": [
    {"draft_id": "dr_wo_001", "capability": "eam.workorder.draft.create", "summary": "停机维修工单 PRESS-03 / E-204 / 预计 45 分钟", "requires_confirmation": true, "requires_approval": true},
    {"draft_id": "dr_req_001", "capability": "erp.spare.requisition.draft", "summary": "领料 溢流阀 XX-501 x1", "requires_confirmation": true},
    {"draft_id": "dr_dis_001", "capability": "shift.dispatch.draft", "summary": "派工 王五（液压L2，对讲机CH7）", "requires_confirmation": true}
  ],
  "incident_id": "inc_20260425_0815_PRESS03_E204"
}
```

**UI 要求**：
1. `safety_alert` 类型必须置顶展示，使用 `SafetyAlert` 组件
2. `sources` 中的 `safety_regulation` 类型需要特殊标识（红色警示图标）
3. `draft_actions` 多个草稿需要支持「一键确认全部」
4. `locator` 字段支持 SOP 步骤号格式（如 `ch5.4`、`§3.1`、`wo:00801`）
