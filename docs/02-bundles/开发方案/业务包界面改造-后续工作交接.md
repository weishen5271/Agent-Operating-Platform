## 1. 阶段 2 — 多包叠加与意图路由 UI（§7）

### 1.1 目标

让 Shell 能同时承载「主行业包 + 多通用包」，PackageContextBadge 显示叠加数，意图路由结果在对话页解释为什么本次回答归属 A 包。

### 1.2 后端任务

#### 任务 2.1 租户多包配置 API

**文件**：
- 新增 [api/routes/admin.py](../../apps/api/src/agent_platform/api/routes/admin.py)（在 `/packages` 附近增端点）
- [runtime/chat_service.py](../../apps/api/src/agent_platform/runtime/chat_service.py)（新增 `list_tenant_packages` 方法）
- [domain/models.py](../../apps/api/src/agent_platform/domain/models.py)（`TenantProfile` 现仅 `package: str` 单值，需扩为 `primary_package: str` + `enabled_common_packages: list[str]`）

**契约**：
```python
# 新端点 GET /admin/tenants/{tenant_id}/packages
# 响应：
{
    "primary_package": "industry.hr",
    "common_packages": ["_common/report_gen", "_common/data_qa"],
    "available_packages": [
        {"package_id": "industry.hr", "name": "HR 业务包", "domain": "industry"},
        {"package_id": "_common/report_gen", "name": "报告生成", "domain": "common"},
        ...
    ]
}

# 新端点 PUT /admin/tenants/{tenant_id}/packages
# 入参：{primary_package, common_packages: [...]}
```

**注意**：`TenantProfile` 的 `package` 字段当前在 [TenantRepository](../../apps/api/src/agent_platform/infrastructure/repositories.py) 持久化为单字段。需 alembic 迁移加 `enabled_common_packages JSONB DEFAULT '[]'`，并保持 `package` 作为 primary 的兼容读路径。

#### 任务 2.2 路由器升级

**文件**：[runtime/chat_service.py](../../apps/api/src/agent_platform/runtime/chat_service.py) — `_build_routing_decision` 当前只看 intent，需要：

1. 读取租户的 `primary + commons` 列表
2. 把 commons 作为候选，按 intent → package 映射打分
3. 输出 `candidates: [{package_id, confidence}]` 至少 1-2 条

仍可保持 in-memory 启发式，等 PackageLoader 接入后再抽 router 类。

集成位置：[components/shared/shell.tsx](../../apps/web/src/components/shared/shell.tsx) 顶部栏 apps 图标弹层（line 215 附近 console-header-left）。

## 5. 工程纪律（必须遵守）

来自既定方案，落地时不得绕过：

1. **不在前端拼装 yaml**：所有写操作经后端 API，yaml 仍是真源
2. **行业字段全部走 metadata**：`mfg_*` 字段不进通用组件 props，未知字段进「更多属性」折叠区
3. **每章先发只读版**，受控编辑（Tool 配额覆盖、插件配置、Skill 回滚）单独评审
4. **密钥字段绝不允许 `<input type="text">`**：组件层强校验，违反则拒渲染
5. **灰度先发自己**：UI 改造按 feature flag (`ui.package_v2`) 灰度租户
6. **保留 `draft_action` 兼容字段**：草稿包是新增 `draft_action_group`，不破坏老页面
7. **风险颜色继续用现有 `risk-level`**：禁止新增视觉体系
8. **测试同步更新**：每改 ChatService 构造签名，记得改 [test_chat_runtime_chain.py](../../apps/api/tests/test_chat_runtime_chain.py) line 178 + [test_rag_quality.py](../../apps/api/tests/test_rag_quality.py) line 84
