# Local Codex Instructions

## 关键约束
- 不要随便兜底。
- 不要在未确认必要性的情况下直接实现“看起来可能有用”的方案。
- 先定位根因，再决定是否需要实现、兼容或降级处理。
- 如果存在明显取舍或风险，先明确说明假设与影响，再继续执行。
- 增加列字段的的时候，增加变更语句，而不是去修改代码逻辑
- 未经用户明确允许，不得为了开发、演示、联调、补齐流程或通过校验而伪造业务数据、测试结果、接口返回、数据库记录、日志内容或其他会被误认为真实的信息。
- 不要把业务配置、凭据、租户数据、接口返回或环境差异写成硬编码；如确需临时占位或模拟，必须先提示用户、明确标注为“模拟/占位”，并说明未来应在哪个阶段替换为真实来源。
- 新增或调整后端 API 时，必须同步维护中文 OpenAPI 元数据，包括 tag 映射、接口 summary/description、请求参数和请求体字段 description，确保 Apifox/Swagger 导入后中文可读。
